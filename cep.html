Comunicación Entre Procesos

IPC · Inter-Process Communication · Animación Interactiva

▸ ¿Qué es la Comunicación entre Procesos (IPC)?

La Comunicación entre Procesos (IPC — Inter-Process Communication) es el conjunto de mecanismos que permite a procesos que se ejecutan en distintas máquinas o en la misma máquina intercambiar datos y coordinarse. En sistemas distribuidos, donde múltiples procesos corren en nodos separados conectados por una red, la IPC es la columna vertebral que hace posible la colaboración entre componentes.

▸ Middleware como puente

Según Sosa (2017), el Middleware es un software de conectividad que consiste en un conjunto de servicios que permiten interactuar a múltiples procesos que se ejecutan en distintas máquinas a través de una red. Oculta la heterogeneidad, abstrae la complejidad subyacente y provee un modelo de programación conveniente. La IPC es el mecanismo interno que el middleware expone al desarrollador.

▸ Tipos de Comunicación en Sistemas Distribuidos

Los sistemas distribuidos implementan distintos patrones de comunicación según las necesidades de acoplamiento, latencia, confiabilidad y escalabilidad requeridas por la aplicación.

📨

Paso de Mensajes (Message Passing)

Mecanismo básico donde los procesos intercambian datos encapsulados en mensajes. Pueden ser síncronos (bloqueantes) o asíncronos (no bloqueantes). Ejemplos: sockets TCP/UDP, MPI.

☎️

RPC — Llamada a Procedimiento Remoto

Permite a un proceso invocar una función en otro nodo como si fuera local. El stub cliente serializa los parámetros (marshalling), los envía, y el stub servidor los deserializa y ejecuta.

📬

Colas de Mensajes (Message Queues)

Canal asíncrono con buffer persistente. El emisor publica mensajes sin esperar al receptor. Alta tolerancia a fallos. Ejemplos: RabbitMQ, Apache Kafka, AWS SQS.

📡

Publicar/Suscribir (Pub-Sub)

Los emisores (publishers) publican eventos en tópicos. Los suscriptores (subscribers) reciben solo los eventos de los tópicos que les interesan. Bajo acoplamiento entre componentes.

🌐

Servicios Web (REST / SOAP)

Comunicación basada en HTTP. REST usa JSON/XML sobre URLs. SOAP usa un protocolo estándar XML. Son la base de las APIs modernas y la integración de microservicios en la nube.

🔗

Streams y Eventos

Comunicación continua mediante flujos de datos en tiempo real. Los procesos reaccionan a eventos conforme ocurren. Ejemplos: WebSockets, gRPC streaming, Apache Flink.

▸ Comunicación Síncrona vs. Asíncrona

Esta es la distinción más importante en el diseño de sistemas distribuidos. La comunicación síncrona bloquea al emisor hasta recibir respuesta. La asíncrona permite al emisor continuar trabajando mientras el mensaje se procesa en segundo plano.

Sockets TCP/UDP SÍNC / ASÍNC Alto Muy baja TCP: Alta / UDP: Baja Comunicación en tiempo real, videojuegos

▸ Cloud Computing y la Web como Middleware

El Cloud Computing es el modelo que ofrece recursos de cómputo, almacenamiento y redes como servicios bajo demanda a través de Internet. La web actúa como middleware universal, permitiendo que procesos en cualquier parte del mundo se comuniquen usando protocolos estándar (HTTP, WebSocket, MQTT).

☁️

IaaS — Infraestructura como Servicio

Se alquilan máquinas virtuales, redes y almacenamiento. El usuario gestiona SO y aplicaciones. Ej: AWS EC2, Azure VMs, Google Compute Engine.

🏗️

PaaS — Plataforma como Servicio

Se provee el entorno de ejecución completo. El desarrollador solo despliega código. Ej: Heroku, Google App Engine, AWS Elastic Beanstalk.

📦

SaaS — Software como Servicio

Aplicaciones completas accesibles desde el navegador. El proveedor gestiona todo. Ej: Gmail, Salesforce, Microsoft 365, Zoom.

// ARQUITECTURA CLOUD: WEB COMO MIDDLEWARE

▸ La Web como Middleware Universal

La IETF definió middleware como un conjunto de servicios y funciones reutilizables, expandibles, utilizadas por muchas aplicaciones para funcionar dentro de un ambiente interconectado. La Web cumple exactamente ese rol: HTTP es el protocolo universal de comunicación entre procesos distribuidos en la nube. Los servicios REST, SOAP, GraphQL y gRPC son las interfaces que exponen los procesos para comunicarse sobre esta infraestructura.
